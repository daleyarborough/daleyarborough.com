  • VPC | Amazon Virtual Private Cloud

    • Private IP Address Range CIDR -- 10.0.0.0/16

    Three Private Address Ranges -- 10.0.0.0/0 (10/8 prefix), 172.16.0.0 (172.16/12) prefix, 192.168.0.0 (192.168/16 prefix)

    Two routes of entry -- Internet Gateway (how we connect to internet), Virtual Private Gateway (where we terminate our VPN connections) Where we can have site-to-site connection between a VPC and a datacenters.

    The router then routes to AWS Route Tables, followed by Network Access Control Lists (referred to as second line of defense),
    and finally into the different subnets, whether it is a Private Subnet or a Public Subnet.
  • ONE Subnet can only span ONE Availability Zone, but Security Groupds, Route Tables, Network ACL's can span multiple Availability Zones

    • You will then launch your EC2 instances inside these Subnets, whether DB or application servers in private, or web servers in public

  • Network Access Control Lists are stateless, meaning that if you allow traffic in from port 80 (http) or 443 (https),
    then you must also create a rule allowing it back out again
  • Each EC2 Instance comes with a Private IP and a Public IP by default. VPC Subnets default to having route OUT to internet
  • VPC Peering | Allows multiple VPC's in a Region to connect and communicate them through Direct Connect via Private IP's
  • VPC is in "Star Configuration", meaning there is no such thing as "Transitive Peering" IE : Each VPC connections has to be set up seperately! Not a middle VPC that talks to all
  • Security Groups are STATEFUL, Network Control Access Lists (NACL) are STATELESS

  • An Internet gateway is a horizontally scaled, redundant, and highly available VPC component that allows
    communication between instances in your VPC and the Internet. It therefore imposes no availability risks or bandwidth constraints on your network traffic. An Internet gateway serves two purposes: to provide a target in your VPC route
    tables for Internet-routable traffic, and to perform network address translation (NAT) for instances that have been assigned public IPv4 addresses
    To use an Internet gateway, your subnet's route table must contain a route that directs Internet-bound traffic to the Internet gateway
