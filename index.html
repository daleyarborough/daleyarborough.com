S3 (Simple Storage Service)


is a place to store files in the cloud. Object based storage (flat files).
Designed to widthstand failure, with high availability and durability. Files can be from 1Byte up to 5TB.
Files are stores in "buckets" Read and Write access for PUTS of NEW objects; Eventual consistency for overwrite PUTS and DELETES
Built for 99.99% availability for the S3 platform. Amazon guarantees 99.99% availability; 99. (11 x 9's) durability
Tiered storage available. Lifecycle Management. Versioning. Encryption. Secure data with "bucket policies and access control lists.
S3, S3-IA, Glacier (long retrieval time, usually 3-5 hours | used generally for data archival,
RRS (use case: data you can generate again -- image files in one bucket, and thumbnails in another (RRS)
Things charged for when using S3: Storage, Requests, Storage Management Pricing, Data Transfer Pricing,
Transfer Acceleration File Gateway - for flat files, stored directly in S3 Volume Gateway
